## 1. 项目概述

一款面向乒乓球发烧友的微信小程序，帮助用户发现免费乒乓球场地、查看实时人数、约球交友。

**目标用户：** 乒乓球爱好者，尤其是想找免费场地打球、寻找球友的人群。

---

## 2. 核心需求

### 2.1 地图选点 + 导航

用户打开小程序后，在地图上查看附近的免费乒乓球场地，点击可查看场地详情，并一键唤起导航。

- **可行性与成本：** ⭐⭐⭐⭐⭐ 非常高
- 微信原生 `<map>` 组件 + `wx.openLocation()` 即可实现
- 完全免费，无第三方 API 费用

### 2.2 实时人数统计

授权位置后，小程序显示各场地当前打球人数。

- **可行性与成本：** ⭐⭐⭐⭐ 签到模式可行
- 自动检测（传感器/探针/信令）不可行
- 推荐「用户主动签到」模式

## 5. 核心功能技术方案

### 5.1 地图选点 + 导航

**技术栈：** 微信原生 `<map>` 组件 + `wx.openLocation()`

**关键 API：**

| API | 用途 |
|-----|------|
| `wx.getLocation()` | 获取用户位置 |
| `<map>` 组件 | 渲染地图，markers / callout / controls |
| `wx.openLocation()` | 唤起微信内置导航 |
| `mapContext.includePoints()` | 自动缩放适配所有 marker |

**附近场地查询：**

```sql
-- MySQL 8.0 地理空间查询
SELECT *, ST_Distance_Sphere(point(lng, lat), point(?, ?)) AS distance
FROM courts
WHERE ST_Distance_Sphere(point(lng, lat), point(?, ?)) < 5000
ORDER BY distance;
```

```bash
# Redis GEO（更高性能）
GEOADD courts 116.397 39.908 "朝阳公园"
GEORADIUS courts 116.397 39.908 5 km WITHDIST ASC
```

### 5.2 实时人数统计

**技术栈：** Redis + WebSocket

```
用户点击"开始打球"
       │
       ▼
  GPS 校验（200m 内）
       │
  ┌────┴────┐
  │ 通过     │ 拒绝 → 提示"靠近场地后才能签到"
  └────┬────┘
       ▼
  Redis INCR court:123:active         ← 人数 +1
  Redis SADD court:123:users uid      ← 记录用户
  Redis SETEX user:uid:checkin court_id 10800  ← 3h 超时
       │
       ▼
  WebSocket 广播人数变更
```

**实时同步方案对比：**

| 方案 | 实时性 | 成本 | 适用 |
|------|--------|------|------|
| **WebSocket** | 秒级 | 需维护长连接 | ✅ 首选 |
| 轮询（30s） | 半分钟 | 简单但带宽浪费 | 备选 |
| 云开发 watch | 秒级 | 微信云开发 | 小项目 |

**防作弊策略：**

| 策略 | 说明 |
|------|------|
| GPS 围栏 | 超出 200m 自动签退 |
| 加速度计 | 检测手机是否有运动特征 |
| 蓝牙辅助 | 未来可部署 Beacon 设备到场地 |
| 异常检测 | 频繁签到/签退 → 限流风控 |
| 超时退出 | 3 小时无操作自动签退 |

### 5.3 约球广场 + 即时组局

**技术栈：** RESTful API + 微信订阅消息

**数据模型（简化）：**

```
posts 表:
  id, user_id, court_id, title, level,
  player_count, joined_count,
  start_time, status(招募中/已满/已结束), created_at

post_joins 表:
  id, post_id, user_id, status(申请/已通过/已拒绝)
```

**通知链路：**

```
有人申请加入 → 推送模板消息给发起人
发起人通过   → 推送模板消息给申请人
开场前 30 分钟 → 推送提醒给所有参与者
```

### 5.4 成就系统 + 排行榜

**技术栈：** Redis Sorted Set + 定时任务

```bash
# 排行榜
ZADD rank:weekly:checkin 15 user:1001   # 本周打卡 15 次
ZADD rank:monthly:checkin 62 user:1001  # 本月打卡 62 次
ZREVRANGE rank:weekly:checkin 0 99 WITHSCORES  # Top 100

# 每周一凌晨重置（定时任务）
DEL rank:weekly:checkin
```

**成就触发架构：**

```
用户行为事件 → 事件总线 → 规则引擎 → 达成条件? → 发放成就
```

**规则配置化（JSON）：**

```json
{
  "achievement_id": "visit_100_days",
  "name": "百日球王",
  "condition": { "metric": "total_checkin_days", "op": "gte", "value": 100 },
  "reward": { "type": "badge", "icon_url": "xxx" }
}
```

### 5.5 内容 + 媒体

**技术栈：** 腾讯云 COS + 云点播 VOD

```
用户上传 → 小程序端压缩 → 服务端签发临时密钥
    → 客户端直传 COS（不经服务器）
    → COS 回调 → 后端记录 URL → 内容安全审核
```

| 存储类型 | 方案 | 说明 |
|----------|------|------|
| 图片 | 腾讯云 COS + 万象图片处理 | 自动压缩、裁剪、水印、WebP |
| 视频 | 腾讯云 VOD | 转码、防盗链、播放器 SDK |
| CDN | 腾讯云 CDN | 加速分发 |

### 5.6 搜索 + 球友匹配

**初期方案（MySQL 替代）：**

```sql
-- MySQL 全文搜索场地
SELECT * FROM courts
WHERE MATCH(name, address) AGAINST('朝阳 免费' IN BOOLEAN MODE)
AND ST_Distance_Sphere(...) < 5000;
```

```bash
# Redis GEO 做附近球友
GEORADIUS players:geo 116.397 39.908 10 km
```

**规模化方案（Elasticsearch）：**

```
ES 索引: courts
  - name (分词), address (分词)
  - facilities (标签: 灯光/室内/免费)
  - location (geo_point), rating (评分)

ES 索引: players
  - style (打法), level (水平)
  - preferred_time (偏好时段)
  - location (geo_point), activity_score (活跃度)
```

## 6. 数据库设计

### 6.1 核心表结构（MySQL）

```sql
-- 用户表
CREATE TABLE users (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    openid VARCHAR(64) NOT NULL UNIQUE,
    nickname VARCHAR(64),
    avatar_url VARCHAR(512),
    style VARCHAR(32),           -- 打法: 横拍弧圈/直拍快攻/削球/...
    level TINYINT DEFAULT 1,     -- 水平: 1初级 2中级 3高级
    city VARCHAR(64),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 场地表
CREATE TABLE courts (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(128) NOT NULL,
    address VARCHAR(512),
    lng DECIMAL(10,7) NOT NULL,
    lat DECIMAL(10,7) NOT NULL,
    table_count TINYINT DEFAULT 1,    -- 球台数量
    surface VARCHAR(32),              -- 地面材质
    has_light TINYINT DEFAULT 0,     -- 是否有灯光
    is_indoor TINYINT DEFAULT 0,     -- 是否室内
    is_free TINYINT DEFAULT 1,       -- 是否免费
    open_time VARCHAR(64),            -- 开放时间描述
    photos JSON,                      -- 照片 URL 数组
    status TINYINT DEFAULT 1,         -- 1正常 0已拆除
    contributor_id BIGINT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    SPATIAL INDEX geo_idx (pt),
    FULLTEXT INDEX ft_idx (name, address)
);

-- 评价表
CREATE TABLE reviews (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    court_id BIGINT NOT NULL,
    rating TINYINT NOT NULL,          -- 1-5 评分
    content TEXT,
    photos JSON,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 签到记录表
CREATE TABLE checkins (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    court_id BIGINT NOT NULL,
    start_time DATETIME NOT NULL,
    end_time DATETIME,
    duration INT DEFAULT 0,           -- 打球时长(分钟)
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 收藏表
CREATE TABLE favorites (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    court_id BIGINT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_court (user_id, court_id)
);

-- 约球帖子表
CREATE TABLE posts (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    court_id BIGINT NOT NULL,
    title VARCHAR(128),
    level TINYINT,
    player_count TINYINT DEFAULT 2,   -- 需要人数
    joined_count TINYINT DEFAULT 0,   -- 已加入人数
    start_time DATETIME,
    status ENUM('recruiting','full','finished','cancelled') DEFAULT 'recruiting',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 约球参与表
CREATE TABLE post_joins (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    post_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    status ENUM('pending','approved','rejected') DEFAULT 'pending',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 战绩表
CREATE TABLE matches (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    winner_id BIGINT NOT NULL,
    loser_id BIGINT NOT NULL,
    court_id BIGINT,
    winner_score INT DEFAULT 0,
    loser_score INT DEFAULT 0,
    played_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 成就表
CREATE TABLE achievements (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    achievement_key VARCHAR(64) NOT NULL,
    unlocked_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_achievement (user_id, achievement_key)
);
```

### 6.2 Redis 数据结构

```bash
# 实时人数
court:{id}:active_count      → INT      # 当前打球人数
court:{id}:active_users      → SET      # 当前在场用户 ID

# 签到状态
user:{id}:checkin            → STRING   # 签到所在场地 ID (带 TTL)
user:{id}:checkin_time       → INT      # 签到时间戳

# 排行榜
rank:weekly:checkin          → ZSET     # 周打卡榜
rank:monthly:checkin         → ZSET     # 月打卡榜
rank:total:checkin           → ZSET     # 总榜
rank:weekly:contribute       → ZSET     # 周贡献榜

# 附近用户
players:geo                  → GEO      # 用户位置（用于球友匹配）

# 限流
rate:user:{id}:checkin       → INT      # 签到频率限制（带 TTL）
```
